Skin files

Registered and shareware users can use skin files to customize the screen saver look. You can use these sample files or create your own pictures. Any object you will use will move according the gravity laws, only the look is changed. These screenshots have been reduced to optimize the download of this page. The two following skins are included with the installation program.

To use a skin file:

Download the file, extract it into the directory of you choice ( the description file and the bitmap files should be in the same directory ).

  • Open the Gravitation configuration window.
  • Go to the 'Display' section.
  • Check the 'Use skin file' option
  • Enter the path to the skin file into the 'Skin file' option.

How to create a skin file:

The easiest way to create your own skin file is to start from the one provided with the installation program.

A 'skin file' is composed from various files:

  • A description text file
  • Severall picture files
  • A palette file

In the following, I will assume that you can use a text editor and a graphic editor. You must also know picture formats, color palettes...

Description file:

This is the text file that helps Gravitation find other files and some option. Following is a sampel description file:

[Skin] Must be the first line of the description file
CrossHair color=135 Palette index (0..255) to display the crosshair
Comet image=-1 Index of the image that have comet tail. Use -1 if no one of your bitmaps is a comet
Nb background images=0 Number of pictures that will be displayed in the background (1..16)
Nb Images=5 Number of planet images (1..16)
Transparency=64 Palette of the transparency part of the pictures (0.255) (the palette color used for the transparency is also the color of the background ).
Palette=Renaissance\PaletteCuivre.dib Name of the palette file that contains all the 255 colors. I suggest you use the one provided with the installation programm and change the palette keeping color indexes to create your own.
Explosion=Renaissance\explosion.dib Name of the picture used to display explosions when planets collide.
Planet 0=Renaissance\Sputnik.dib Name of a picture for a planet. This picture is a horizontal collection of square picture. Each square is a frame from the planet animation. You can have as many frames as you want, within memory limits. The picture file must be a 256 colors windows DIB file. Transparent parts of the image must be colored with the palette index specified with the transparency option.
Planet 1=Renaissance\Fusee.dib Name of the second planet picture. You can have up to 16 planet pictures.
Background 0= Name of the first background picture. The background pictures must use be Windows 256 colors DIB files and use the same color palette as the planet files. Should I say, all picture from a skin file must use the same palette, I'm sure you already know that.
Background 1= Name of the second background picture. You can have up to 16 background pictures
Nb Star colors=10 Number of palette colors used to display the background stars (0..20).
Star color 0=246 One of the color indexes used to display the background stars.
Nb Comet colors=5 Number of palette colors used to display the comet tails (0..20).
comet color 0=239 One of the color indexes used to display the comet tail.

 

Creating the picture files:

- Draw all you picture frame into separate files. Each picture must be square and have the same size.

- Create a large file containing all your frames horizontally. You can find in the downloads section a utility that will help you to paste your frame into a single band.

- Apply the global palette to your picture.

- Save the picture into a 256 colors Windows DIB file.
